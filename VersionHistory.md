## Companion Specs

Some entries in the version table are versioned independently of the core A2AL spec. They use a prefix:

- Core spec: bare version (`0.3.0`, `0.2.x`)
- Companion: `<name>/<version>` (e.g., `a2a-shorthand/0.1.0`)

Profiles registered under A2AL itself (e.g., `project-coord/1.0`, `social-post/1.0`) are documented in [`profiles/PROFILES.md`](./profiles/PROFILES.md) and are not separately listed here.

## Versioning Policy

A2AL follows semantic versioning. The core spec and each profile version independently.

- **MAJOR (x.0.0)** — breaking change to envelope shape, type bans, or core validation rules
- **MINOR (0.x.0)** — new optional fields, profiles, intents, or section types
- **PATCH (0.0.x)** — clarifications and validator fixes

Pre-1.0 means breaking changes are expected. /1.0 is reached when at least one production agent emits and consumes A2AL successfully.

A profile's version (e.g. `project-coord/1.0`) is independent from the core spec version.
